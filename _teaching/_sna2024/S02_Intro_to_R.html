Functions in R

Functions are central in R We utilize them to perform operations on our data

?round # calls help page for function round()
starte den http Server für die Hilfe fertig

round(x, digits = 0)

“round rounds the values in its first argument to the specified number of decimal places (default 0). See ‘Details’ about “round to even” when rounding off a 5.”

Default Values:

Functions often have multiple parameters. In this case x, the number to be rounded, and digits, the number of decimal places. Often, functions have default values, which means that some parameters have a value they take when they are unspecified. All other values need to be specified!

round(x = 12.345) # defaults to 0 decimal places
[1] 12
round(x = 12.345, digits = 0) # explicitly states 0 decimal places
[1] 12
round(x = 12.345, digits = 1) # asks for 1 decimal place
[1] 12.3
# round(digits = 0) # argument without default value is unspecified -> error

If you don’t explicitly state what element of a function you want to call, R assumes them in the order of the function

round(x = 12.345, digits = 0) # explicitly stating each parameter
[1] 12
round(12.345, 0) # letting r assume the default order of all parameters
[1] 12
round(digits = 0, x = 12.345) # now, we don't need to adhere to the default order of parameters
[1] 12
round(0, 12.345) # here, r incorrectly assumed the standard order of parameters and rounded 0 to 12.345 decimal places
[1] 0

Good Practice: always specify parameters explicitly. This may be unnecessary in simple functions, but becomes very important in complex functions!

Some examples of simple functions

sqrt(x = 25) # taking the (square) root
[1] 5
abs(x = -45) # absolute value
[1] 45
log(x = 12) # natural logarithm
[1] 2.484907
exp(x = 2) # exponentiate 
[1] 7.389056

Code Alignment

If you explicitly state each parameter of a function, you can align the code for better readability:

round(x = 12.345, 
      digits = 0)
[1] 12

Good Practice: Again, this may be irrelevant for simple functions, but for more complex function calls with many arguments this is really helpful.

Variables

Using the <- operator, we can assign values to variables

x <- 3

This variable can then be called from storage (the Global environment in the upper right panel):

x
[1] 3

Also, we can perform operations on variables:

y <- x + 3 # set y to x + 3

x / 4 # divide x by 4
[1] 0.75
x <- x + 1 # increase value of x by 1

x # show new value of x
[1] 4
